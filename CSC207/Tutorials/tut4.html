Setup and Submission

  • Launch Eclipse Neon (if you see a blank screen, close the blank tab)
  • Switch to Git perspective (or Window > Show View > Other > Git > Git Repositories) to view a list of all your git repositories in your Eclipse sidebar.
  • Right click on your repo_UTORID.git repo and select "Pull" (if you don't have this directory, follow instructions under the "Git" and "Making Eclipse aware of your repo" from the first lab).
  • File > Import > Git > Projects from git
  • Select "Existing local repository"
  • Select repo_UTORID
  • Select "Import existing Eclipse project" and from the dropdown, select LabWeek05 (the whole directory; don't browse into it). Select "Next", then "Finish"
  • Now switch to Java Perspective and work with the files in the LabWeek05 project
  • As you work on the lab, commit your changes as follows: Right-click LabWeek05, and select Team > Add to Index (this is git add), then right-click again and select Team > Commit.
  • In the "Git Staging" window, enter appropriate commit message, and then click the "Commit and Push" button.
  • SUBMISSION: Remember to "Commit and Push" your project at the end of your lab so we can see the most updated version of your lab work.

    Lab Instructions

    You might think that this is about GUI programming, really it is a way to trick you into learning about interfaces as well as inheritance (extends).

    You should find several Swing examples in the LabWeek05 directory in your repository. This is the same code we worked through during last lecture.

    1. First, as I did in lecture, fix HiBye.java and HiByeActionListener.java as described in the code. Remember, to fix this, you will need to modify the constructor for HiByeActionListener.
    2. Create class PrimeTester inside the LabWeek05 project. PrimeTester should launch the following GUI. Three different application screenshots are shown below.




      This consists of a JTextField, a JButton and another JTextField. The second JTextField is set to be non-editable (take a look at the Java API for a JTextField to see how to set it to be non-editable. For this example, copy HiBye.java to PrimeTester.java and copy HiByeActionListener.java to PrimeTesterActionListener.java.

      To parse an integer and catch errors, use something like the following ...

      try { int n=Integer.parseInt(this.question.getText()); // n is a valid integer in here... } catch(NumberFormatException nfe){ System.out.println("invalid number"); }

      Use the following isPrime method.

      private boolean isPrime(int n){ for(int i=2;i<n;i++){ if(n%i == 0) return false; } return true; }
    3. You are going to re-build the above application, this time, copy UseButtonPanel.java to PrimeTester2.java and copy ButtonPanel.java to PrimeTesterPanel.java. Complete PrimeTester2.java and PrimeTesterPanel.java so that it duplicates the functionality in question 2. Consider yourself doing VERY WELL if you understand how this works!
